# Contributing to AI-Ethics

感謝你對本專案的興趣與貢獻！為了讓協作順暢並兼顧安全與倫理，請依照以下簡要指引貢獻。

1. 以 Issue 開始討論  
   - 若你有功能建議、文件改進或倫理/安全疑慮，請先開 issue 描述背景與動機，方便社群討論與迭代。

2. Pull Request 流程  
   - fork → 建分支 → 開發 → 發 PR。  
   - PR 標題請清楚標示改動範圍（例如：docs: update README trust section）。  
   - 若 PR 牽涉到實驗或模型評估，請在 PR 描述中包含重現步驟、資料來源、與倫理風險評估（見下）。

3. 實驗與資料使用的倫理要求（必讀）  
   - 不得將含有未經授權個資或敏感資料的示例直接公開。  
   - 若實驗可能造成傷害或被濫用，必須在 issue/PR 中明確評估風險並提供緩解措施。  
   - 所有實驗應說明資料來源、授權與匿名化步驟，以及可能的偏差與限制。

4. 文件與範例  
   - 範例程式碼與 Demo 必須包含簡短的 disclaimers：用途、限制、以及必要的人類核驗步驟。  
   - 盡量提供可重現的最小示例（minimal reproducible example）。

5. 回報安全或倫理問題  
   - 若發現高風險或可能違法濫用的情形，請使用 repository 的 Ethics/Safety issue template 回報，或聯絡維護者以獲取私密處理途徑。

6. 行為準則  
   - 尊重他人、保持建設性討論。若發生爭議，維護者將依 CODE_OF_CONDUCT 處理（若未另行包含，會依通用開源社群準則執行）。

感謝你的投入 — 我們希望建立一個既開放又負責任的社群.
